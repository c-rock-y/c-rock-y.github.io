Query string & Simple Query String

Query string

query string 类似于 URI Query

default_field： 查询字段名
query： 查询条件
and: 与操作

Simple Query String

  • 类似于Query String 只支持部分查询语法
  • 不支持 AND OR NOT 会当做字符串处理
  • Term 之间默认的关系是 or 可以指定 default_operator
  • 支持部分逻辑
      • 代替 and
      • 代替 not
    • | 代替 or

Dynamic Mapping 和常见的字段类型

什么是 Mapping

Mapping 类似于数据库中的 Schema 的定义，作用如下

  • 定义索引中的字段的名称
  • 定义字段的数据类型，eg： string int bool
  • 字段，倒排索引及分词的相关配置，

Mapping 会把 JSON 文档映射成 Lucene 所需要的扁平格式。

一个索引的Maping 属于一个索引的 Type

  • 每个文档都属于一个Type
  • 一个 Type 有一个 Mapping 定义
  • 7.0 开始，不需要在 Mapping 定义 type 的信息

字段的数据类型

简单类型

  • Text / Keyword

  • Date

  • Integer / Floating

  • Boolean

  • IPv4 / IPv6

    复杂类型

  • 对象类型/嵌套类型

    特殊类型(针对于地理信息)

  • geo_point & geo_shape / percolator

Elastic

基本概念

文档

最小的可搜索文档

文档被序列成json格式 ，被保存在es当中

  • json对象是有字段组成
  • 每个字段都有对应的字段类型（字符串，数值，布尔，日期，二进制，范围类型）

每一个文档都有一个 Unique ID

  • 可以自己指定
  • 可以通过 elasticsearch 自动生成

json文档

  • 一篇文档包含了一系列的字段，类似数据库当中的一条记录
  • json 文档格式灵活 不需要预先定义格式 （ mysql 不同点）
    • 字段类型可以指定，也可以通过 ElasticSearch 自动推算
    • 支持数组 / 嵌套

文档的元数据

  • 元数据，用于标注文档的相关信息
    • _index: 文档所属的索引名
    • _type: 文档所属的类型名称
    • _id: 文档唯一 id
    • _source: 文档原始 Json 数据
    • _all: 整合所有的字段内容到该字段，7.0 已经废除
    • _version: 文档的版本信息(并发读写时，可以解决文档冲突的问题)
    • _score: 相关性打分

索引

  • Index – 索引是文档的容器，是一类文档的结合
    • Index体现了逻辑空间的概念： 每个索引都有自己的Mapping 定义， 用于定义包含文档的字段名称和字段类型
    • Shard（分片） 体现在物理空间的概念 ：索引的数据分散在Shard上
  • 索引的 Mapping 与Settings
    • Mapping 定义文档字段的类型结构。
    • Setting 定义不同数据分片的分布。

索引不同的含义：

  • 名词： 在同一个集群中，可以创建很多不同的索引。
  • 动词： 保存一个文档到 Elasticsearch 的过程也叫索引（indexing）
  • 名词： 一个B树索引，一个倒排索引 （数据结构）

  • 6.0开始 Type已经被 废弃 ， 7.0开始 一个索引只能创建一个 Type ： “_doc”

抽象和类比

  • 传统关系型数据库和 Elasticsearch 的区别
    • elasticsearch： 全文检索，相关性，对结果进算分
    • RDMS （关系型数据库）： 事务性 和 连表操作
