Elastic

基本概念

分布式系统的可用性和拓展性

  • 高可用性
    • 服务可用性：在集群中有节点服务停止响应的时候，整个集群还能正常的工作（允许有节点停止工作）。
    • 数据可用行： 在集群中有部分节点丢失的时候，数据也不会真正的丢失（部分节点丢失不丢数据v）。
    • 可拓展性
    • 当请求量提升/数据不断增长时，系统又能力把数据分布到节点上，实现水平拓展。

      分布式特性

    • elaticsearch 的分布式架构的好处
      • 存储的水平扩容
      • 提高系统可用性，部分节点停止服务，整个集群的服务不会收到影响
    • elaticsearch 的分布式架构
      • 不同的集群通过不同的名称来区分。默认名称为 “elasticsearch”
      • 通过配置文件的修改，或者在命令行中 -E cluster.name=geektime 来进行设定
      • 一个集群可以有一个或者多个节点。

节点

  • 节点是一个 Elasticsearch 的实例
    • 本质上是一个 JAVA 进程
    • 一台机器上可以运行多个Elasticsearch 进程，但是在生产环境中一般只运行一个 es 实例
  • 每个节点都有名字，通过配置文件配置，或在启动的时候用 -E node.name=node1 指定
  • 每一个节点启动之后，会分配一个 UID 保存在 data 目录下

Master-eligible nodes 和 Master Node

  • 每个节点启动后，默认就是一个 Master-eligible 节点

    • 可以通过设置 node.master: false 来禁止。

  • Master-egligible 节点可以参加选主流程，成为 Master 节点.

  • 当第一个节点启动时，他会将自己选举成 Master 节点。

  • 每个节点上都保存了集群的状态，只有 Master 节点才可以修改集群的状态

    • 集群状态 （Cluster State）， 集中维护了一个集群中必要的信息
      • 所有节点的信息
      • 所有的索引 和 其相关的Mapping 和 setting 信息
      • 分片的路由信息
    • 任意节点都能修改信息会导致数据的不一致性

      Data Note 和 Coordinating Node

  • Data Node

    • 可以保存 es 数据的节点叫做Data Node ，负责保存分片数据。在数据拓展上起了至关重要的作用

  • Coordinating Node

    • 负责接受 client 的请求 ，将请求分发到合适的节点， 最终把结果汇集到一起
    • 每个节点默认都起到 Coordinating Node 的职责

      其他节点类型

  • Hot & Warm Node

    • 不同硬件配置的 Data Node 用来实现 Hot（热数据）&warm（冷数据） 架构 ，降低集群部署的成本 （日志）

  • Machine Learning Node

    • 负责跑机器学习的 Job 用来做异常检测

  • Tribe Node

    • （5.3 开始使用 Cross Cluster Search） Tribe Nede 链接到不同的 Elasticsearch 集群， 并且支持将这些集群当成一个单独的集群处理 （将废弃）

      配置节点类型

  • 开发环境中一个节点可以承担多种角色

  • 生产环境中，应该设置第一的角色的节点 （dedicated node）

    图片

分片

  • 主分片： 用以解决数据水平拓展的问题， 通过主分片， 可以将数据分布到集群内的所有节点上
    • 一个分片是一个运行的 Lucene 的实例
    • 主分片数在索引创建时指定，后续不允许修改，除非 Reindex
  • 副本： 用以解决数据v高可用的问题，分片是主分片的拷贝
    • 副本分片数，可以动态调整
    • 增加副本数，还可以一定程度上提高服务可用性（读取的吞吐）

Mapping 设置

为了能够将时间域视为时间，数字域视为数字，字符串域视为全文或精确值字符串， Elasticsearch 需要知道每个域中数据的类型。这个信息包含在映射中。
索引中每个文档都有 类型（Type） 。每种类型都有它自己的 映射（mapping） ，或者 模式定义 。映射定义了类型中的域，每个域的数据类型，以及Elasticsearch如何处理这些域。映射也用于配置与类型有关的元数据。

当你索引一个包含新域的文档–之前未曾出现– Elasticsearch 会使用 动态映射 ，通过JSON中基本数据类型，尝试猜测域类型，使用如下规则：

JSON type 域 type
布尔型: true 或者 false boolean
整数: 123 long
浮点数: 123.45 double
字符串，有效日期: 2014-09-15 date
字符串: foo bar string

这意味着如果你通过引号( “123” )索引一个数字，它会被映射为 string 类型，而不是 long 。但是，如果这个域已经映射为 long ，那么 Elasticsearch 会尝试将这个字符串转化为 long ，如果无法转化，则抛出一个异常。

定义一个Mapping

定义 Mapping 的一些建议

  • 可以参考api手册，纯手写；
  • 为了减少输入的工作量，减少出错概率，可以依照一下步骤
    • 创建一个临时的index，写入一些样本数据
    • 通过访问Mapping api 获取该临时文件的动态 Mapping 定义
    • 修改后用，使用该配置创建的的索引
    • 删除临时索引

常用参数

  • index —— 控制当前的字段是否被索引，默认为true 如果设置成false，该字段不可被搜索。

    Index Options 4种不同级别的 Index Options 配置，可以控制倒排索引的记录的内容

级别 含义
freqs 索引 Doc id 和 term frequencies
positions Doc id, term frequencies, 和 term positions (or order)
offsets Doc id, term frequencies, positions, and start and end character offsets which map the term back to the original string 
Text 类型默认记录 postions 其他默认 docs
内容储存越多，占用储存越大
  • null_value
    • 可以对null值搜索，
    • 只有Keyword
  • copy_to ： 非嵌套同级聚合多个字段
    • all 在7中已经被 copy_to 所代替
    • copy_to 将字段的数值拷贝到目标字段，实现类似_all的作用
    • copy_to 的目标字段不会出现在_source 中

数组类型

ES 不提供数组类型，但是每种数据类型都可以包含多个相同类型的数值

查看映射

通过 /_mapping ，我们可以查看 Elasticsearch 在一个或多个索引中的一个或多个类型的映射 。在 开始章节 ，我们已经取得索引 gb 中类型 tweet 的映射：

GET /gb/_mapping/tweet

Elasticsearch 根据我们索引的文档，为域(称为 属性 )动态生成的映射。

分词

什么是分词

Analysis 也叫分词，就是将文本转换成一系列单词的过程。

Analysis 与 Analyzer
  • Analysis – 文本分析是吧全文本转换成一系列单词（term/token）的过程，也叫分词
  • Analysis 是通过 analyzer来实现的
    • 可以使用 elasticsearch 内置的分析器、或按需求定制化分析器
  • 除了在数据写入时转换词条，匹配 Query 语句时候也需要相同的分析器对查询语句进行分析

Analyzer 的组成与工作机制

分词器是专门处理分词的组件，Analyzer 由三部分组成

  • Character Filters（针对原始文本处理，例如去除 html）/ Tokenizer(按规则切分单词)/Token Filter (将切分的单词进行加工，小写，删除 stopwords，增加同义词)

Elasticsearch 内置的一些分词器

使用 _analyzer API
Elasticsearch 的内置分词器
  • Standard Analyzer – 默认的分词器按词接分，小写处理
  • Simple Analyzer – 按照费字幕切分（符号被过滤），小写处理
  • Stop Analyzer – 小写处理，停用次过滤（the，a，is）
  • whitespace Analyzer – 按照空格切分，不转小写
  • Keyword Analyzer – 不分词，直接将输入当做输出
  • Patter Analyzer – 正则表达式， 默认 \w+(非字符分割)
  • language – 提供30 多种常见语言的分词器

多字段特性

Es 并没有提供数组类型，但是es支持在字段下面添加多个子字段，例如

  • 添加一个 keyword 类型 ，实现精确搜索
  • 添加不同的语言使用不同的分词进行不同语言，甚至拼音的搜索
  • 可以为搜索和索引指定不同分词

精确值 和 全文本

精确值

  • 包括数数字 / 日期 / 一个具体的字符串（app store）；Es 中的keyword类型

    全文本

  • 全文本，非结构化的文本信息； es中的Text

主要区别 ：精确值不需要被分词，而全文本需要被分词；

自定义分词

当Es自带的分词器无法满足时，可以自定义分词器，通过组合不同的组件实现

分词器的构成

  • Character Filter
  • Tokenizer
  • Token Filter

Character Filter

  • 在 Tokenizer 之前对文本进行处理，例如添加，删除以及替换字符串，可以配置多个 Character Filter，会影响 Tokenizer 的 postion 和 offset 信息。
  • 自带的 Character Filters
    • HTML strip —— 除 html 标签
    • Mapping —— 字符串的替换
    • Pattern replace —— 正则匹配替换

Tokenizer

  • 对文本按照一定规则进行切分
  • 常用 Tokenizer
    • whitespace 空格切分
    • standard 单词边界切割
    • uax_url_email url email地址进行切分
    • patterm 正则切分
    • keyword 不切分
    • path hierarchy 按路径切分、

      Token Filter

  • 将切分好的单词进行删除，修改，添加
  • 常用 Token filter
    • lowercase 转小写
    • stop 除停用词
    • synonym 添加同义词

基本概念

倒排索引

什么是倒排索引

简单理解

  • 图书：
    • 正排索引 - 目录页
    • 倒排索引 - 索引页
  • 搜索引擎
    • 正排索引 - 根据文档 id 到文档内容和单词的关联
    • 倒排索引 - 单词到文档id 的关系

倒排索引的数据结构

倒排索引包含两个部分

字典

  • 单词词典(Term Dictionary)： 记录所有文档的单词，记录单词倒排列表的关联关系（单词词典一般比较大，可以通过 B+ 树或者 哈希拉链法实现）

Position list

  • 倒排列表（posting List）: 记录了单词对应的文档结合，由倒排索引项组成
    • 倒排索引项
      • 文档id
      • 字频TF - 该单词在文档中出现的次数，用于相关性打分
      • 位置（Position）： 单词在文档中分词的位置，用于语句搜索
      • 偏移：记录单词的开始位置，结束位置实现高亮显示

Elasticsearch 的倒排索引

  • Elasticsearch 的JSON 文档中的每一个字段，都有自己的倒排索引
  • 可以指定对某些字段不做索引
    • 优点：可以节省储存空间。
    • 缺点： 字段无法被搜索。

Dynamic Mapping 和 字段类型

什么是Mapping

Mapping 类似于数据库中的 schema （表结构）的定义，Mapping 不仅告诉 ES 一个 field 中是什么类型的值， 它还告诉ES如何索引数据以及数据是否能被搜索到。当你的查询没有返回相应的数据， 你的mapping很有可能有问题。当你拿不准的时候， 直接检查你的mapping。

常见的数据类型

  • 简单类型
    • text/keyword
    • date
    • integer/floating
    • boolean
    • ipv4 / ipv6
  • 复杂类型
    • 对象类型/嵌套类型
  • 特殊类型
    • geo_point & geo_shape / percolator(经纬度类型)

什么是 Dynamic Mapping(动态映射)

Dynamic Mapping（动态映射）是 Elasticsearch的一个重要特性: 不需要提前创建index、定义mapping信息和type类型, 你可以 直接向ES中插入文档数据时, ES会根据每个新field可能的数据类型, 自动为其配置type等mapping信息, 这个过程就是动态映射(dynamic mapping).

但是，有时这是想要的行为有时又不希望这样。通常没有人知道以后会有什么新字段加到文档，但是又希望这些字段被自动的索引。也许你只想忽略它们。如果Elasticsearch是作为重要的数据存储，可能就会期望遇到新字段就会抛出异常，这样能及时发现问题。

幸运的是可以用 dynamic 配置来控制这种行为 ，可接受的选项如下：

值 含义
true 动态添加新的字段–默认
false 忽略新的字段(新增字段无法被搜索)
strict 如果遇到新字段抛出异常

总结

  • Mapping 定义了索引中字段的名称，类型，分词等相关配置。
  • Mapping 会把 json 文档映射成 lucene 所需要的扁平格式。
  • 一个 Mapping 属于一个 type
  • mapping一旦创建之后，就无法修改，只能追加，如果要修改，就需要删除掉整个文档进行重建。
  • Dynamic Mapping（动态映射）:向ES中插入文档数据时, ES会根据每个新field可能的数据类型, 自动为其配置type等mapping信息,

  • ignore_unavailable: 是否忽略单个index是否可用(不存在或关闭), true表示忽略, false表示不忽略, 默认为false, 例如查询已经关闭的index:

Index Template

Index Template：帮助设定 Mapping 和 setting 。并按照一定的规则自动匹配到新创建的索引上

  1. 模板只在创建索引上有效修改模板不会影响已经创建的模板。
  2. 可以设置多个索引模板，设置将会被 merge。
  3. 可以指定 order 的数值控制 mergeing 的优先级。

创建索引时，Mapping 和 setting 的优先级

es默认 < order 低的 indexTemplate < order 高的indexTemplate的设定 < 创建索引时指定的setting，mapping

Dynamic Template

根据es 识别到数据类型，结合字段名称来动态的设定字段类型
例如：

  • 所有字符串设定成 keyword
  • is开头的字段都设置成boolean
  • long_开头的都设置成 long 类型

定义在某个索引的 mapping 中
由名称，配置规则（数组），匹配字段的 mapping 组成。
